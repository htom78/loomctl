# Loom Harness Vision Lock

This MVP is the first slice of a larger platform, not a scope reduction.

The target remains a compact multi-user online sandbox development platform with an auditable harness/loop:

- multi-user workspaces through Coder or `loomd`
- online sandbox development through browser IDE/preview links, terminal, persistent home, and isolated containers
- project/task isolation through worktrees and per-run event logs
- Gitea/Forgejo as the default truth layer for issues, PRs, comments, and code
- LiteLLM as the per-user model gateway and routing layer
- a first-party harness loop that records every user message/run comment, effective policy, model action, tool observation, verification, evaluator result, pause/resume, review, and finish decision
- `brain` as the compounding layer that learns from run summaries and proposes skill improvements through reviewable git changes
- human gates for merge, external side effects, permission escalation, and production deployment

## Non-Negotiable Design Constraints

1. The event schema must stay useful to a future web UI, audit view, Gitea comment bridge, and brain scoring job.
2. Tool execution must be isolated behind a runtime boundary so local execution can later become container or remote execution without changing the loop contract.
3. Verification and configured evaluation are gates, not decorations. A run cannot report success if either configured gate fails.
4. Human steering must become durable run/audit state, not transient browser-only chat.
5. The MVP may omit the web UI, but it must not erase the multi-user online sandbox target.
6. The system may start with scripted and command adapters, but the adapter contract must leave room for LiteLLM-backed model routing.

## Future Control Plane Option

`ngaut/agent-git-service` is a tracked candidate for the shared truth/control layer now that the harness, sandbox, and platform-readiness path have a serve-enabled provider seam. It should not replace the first-party harness/loop: the integration boundary is issues, comments, pull requests, Git transport, workspace branch lease evidence, agent identity, and optional wiki/memory surfaces. Keep Gitea/Forgejo as the default near-term provider. The current executors are local, Docker-backed, and Coder-SSH-backed.
